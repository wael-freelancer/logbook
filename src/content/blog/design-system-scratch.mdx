## What Is a Design System?

A design system is a collection of reusable components, guidelines, and tools that standardize UI development. It's not just a component library — it includes tokens, patterns, documentation, and governance.

### The Three Layers

1. **Design Tokens**: Atomic values (colors, spacing, typography)
2. **Components**: Reusable UI building blocks
3. **Patterns**: Composed layouts and interaction flows

## Design Tokens

Tokens are the foundation. Define them once, consume everywhere:

```css
/* tokens.css */
:root {
  /* Colors */
  --color-primary-50: #f0f9ff;
  --color-primary-100: #e0f2fe;
  --color-primary-500: #0ea5e9;
  --color-primary-600: #0284c7;
  
  /* Spacing scale */
  --space-1: 0.25rem;
  --space-2: 0.5rem;
  --space-3: 1rem;
  --space-4: 1.5rem;
  
  /* Typography */
  --font-sans: 'Inter', system-ui, sans-serif;
  --font-mono: 'JetBrains Mono', monospace;
  
  /* Border radius */
  --radius-sm: 0.25rem;
  --radius-md: 0.5rem;
  --radius-lg: 1rem;
}

/* Dark mode tokens */
@media (prefers-color-scheme: dark) {
  :root {
    --color-primary-50: #0c4a6e;
    --color-primary-500: #38bdf8;
  }
}
```

## Component Architecture

Build components that compose well together. Use CSS custom properties for theming:

```css
/* Button component */
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--space-2);
  padding: var(--space-2) var(--space-3);
  font-family: var(--font-sans);
  font-size: 0.875rem;
  font-weight: 600;
  border-radius: var(--radius-sm);
  border: 1px solid transparent;
  cursor: pointer;
  transition: all 150ms ease;
}

.btn-primary {
  background: var(--color-primary-600);
  color: white;
}

.btn-primary:hover {
  background: var(--color-primary-700);
}

.btn-secondary {
  background: transparent;
  border-color: currentColor;
  color: var(--color-gray-700);
}

.btn-secondary:hover {
  background: var(--color-gray-100);
}
```

## React Component Library

Structure your library for maximum reusability:

```javascript
// components/Button.tsx
interface ButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
  children: React.ReactNode;
  onClick?: () => void;
}

export function Button({ variant = 'primary', size = 'md', children, onClick }: ButtonProps) {
  const sizeClasses = {
    sm: 'px-2 py-1 text-sm',
    md: 'px-3 py-2 text-base',
    lg: 'px-4 py-3 text-lg',
  };
  
  return (
    <button
      className={`btn btn-${variant} ${sizeClasses[size]}`}
      onClick={onClick}
    >
      {children}
    </button>
  );
}

// components/Input.tsx
interface InputProps {
  label: string;
  id: string;
  placeholder?: string;
  error?: string;
}

export function Input({ label, id, placeholder, error }: InputProps) {
  return (
    <div className="space-y-1">
      <label htmlFor={id} className="block text-sm font-medium text-gray-700">
        {label}
      </label>
      <input
        id={id}
        placeholder={placeholder}
        aria-invalid={!!error}
        className={`btn btn-secondary w-full ${error ? 'border-red-500' : ''}`}
      />
      {error && <p className="text-sm text-red-600">{error}</p>}
    </div>
  );
}
```

## Documentation with Storybook

Document every component with examples:

```javascript
// stories/Button.stories.js
import { Button } from '../components/Button';

export default {
  title: 'Components/Button',
  component: Button,
};

export const Primary = {
  args: { variant: 'primary', children: 'Primary Action' },
};

export const Secondary = {
  args: { variant: 'secondary', children: 'Secondary Action' },
};

export const Sizes = () => (
  <div style={{ display: 'flex', gap: '1rem' }}>
    <Button size="sm" variant="primary">Small</Button>
    <Button size="md" variant="primary">Medium</Button>
    <Button size="lg" variant="primary">Large</Button>
  </div>
);
```

## Publishing as a Package

Make your design system consumable:

```json
{
  "name": "@myorg/ui",
  "exports": {
    ".": "./dist/index.js",
    "./styles.css": "./dist/styles.css"
  }
}
```

```javascript
// dist/index.js — barrel export
export { Button } from './Button';
export { Input } from './Input';
export { Modal } from './Modal';
export { Card } from './Card';
```

Consumers install and use:

```javascript
import { Button, Input } from '@myorg/ui';
import '@myorg/ui/styles.css';
```

## Governance

A design system needs rules to stay consistent:

- **Component review**: All new components go through a design review
- **Token changes**: Require approval from the design team
- **Versioning**: Use semantic versioning (major for breaking changes)
- **Deprecation policy**: Mark deprecated components with warnings, remove after 2 major versions

A good design system reduces decisions. Developers don't need to choose colors or spacing — they use what's provided and focus on building features.
